## Qwen Added Memories
- UI Overlap Prevention Rule: Whenever modifying or creating any screen UI layout (MainMenu, ShipMenu, MechSelect, Combat, etc.), always verify that all UI elements (panels, buttons, widgets, text blocks) do not overlap. Use the layout system (s(), sx(), sy()) consistently. Check that: 1) Panels don't intersect, 2) Buttons stay within their parent panel bounds, 3) Widgets (WarCommsWidget, status panels, etc.) don't overlap with other UI, 4) Text elements don't overlap with buttons or panels. If adding new elements, adjust existing positions to make room. Always test visually and fix overlaps before considering the change complete.
